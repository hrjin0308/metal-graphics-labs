# 02-Vertex-Buffer
## Vertex Buffer와 삼각형
### Vertex Buffer
3D 모델은 결국 '정점들의 집합'. 3차원 공간의 한 점을 나타내는 정점은 x, y, z 좌표를 가지는데, 이 좌표와 여러 데이터들을 CPU에서 GPU로 보내야함. 이 때 GPU가 바로 접근해서 빠르게 읽을 수 있는 전용 메모리 상자가 필요. 이 상자가 Vertex Buffer(Metal에선 `MTLBuffer`)

### 왜 하필 삼각형??
GPU 하드웨어는 삼각형을 가장 빠르고 효율적으로 그리도록 설계되어 있음.

* **가장 단순**: 2차원 평면을 만들 수 있는 다각형 중 점이 제일 적음.
* **완벽한 평면 보장**: 세 점은 어떤 방향으로 움직여도 무조건 하나의 평면 위에 존재함
* **무한 분할 가능**: 삼각형은 어떻게 쪼개도 삼각형.

모델러들은 보통 사각형(Quad)로 작업하지만 앱으로 가져올 때 Model I/O 프레임워크가 알아서 삼각형으로 쪼개서 가지고옴

### 와이어프레임(Wireframe) 착시
`setTriangleFillMode(.lines)`를 호출하면 속이 꽉 찬 면 대신 선으로만 모델을 그림. 구체모양 와이어프레임 모델을 살펴보면 수많은 삼각형이 모여서 부드러운 곡면처럼 보이는 착시를 일으킴. 이를 이용해서 곡선을 표현.

## Meterial
### Meterial
재료/물질이란 뜻. 빛이 물체 표면과 상호작용하는 방식을 정의하는 수학적/시각적 속성의 집합. 화면 속 물체가 어떤 느낌(금속, 플라스틱, 유리, 피부 등)으로 보일지 결정하는 렌더링의 핵심 요소.

#### Meterial과 Shader  
Meterial과 Shader는 '설계도와 제품' 관계
  * Shader: 빛과 그림자를 어떻게 계산할지 뼈대를 짜놓은 수학적 프로그램(코드). (ex> 빛이 들어오면 거칠기와 금속성을 계산해서 화면에 픽셀을 뿌려라)
  * Meterial: Shader라는 설계도에 실제 값(ex> 파란색, 거칠기 0.2, 금속성 0.9 등)이나 이미지 파일(Texture)을 대입한 결과물

### Meterial 속성
* **Diffuse**: 물체 표면의 가장 기본이 되는 바탕 색상.
* **Metallic**: 이 표면이 금속 재질인지 아닌지를 나타냄. 0~1 값.
* **Roughness**: 거친 정도. 0에 가까울수록 표면이 완전 유리처럼 매끄럽고 빛남.
* **Meterial Group**: 자동차의 '유리창', '바퀴', '차체'처럼 부위별로 다른 색과 재질을 주기 위해 정점들을 나눠놓은 덩어리.

#### PBR Meterial
Physically Based Rendering. '빛은 물리 법칙에 따라 반사되어야 한다'라는 가이드라인이자 시스템.
* Albedo / Base Color: 빛이나 그림자가 전혀 없는 상태의 물체 고유의 색상
* Metalic / Metalness: 물체가 금속인지 비금속인지 결정. 0~1 값. 금속은 주변 환경을 강하게 반사하고 고유의 반사광 색상을 가짐.
* Roughness /Smoothness: 표면의 거친 정도. 표면이 매끄러우면(Low Roughness) 거울처럼 맑게 반사되고, 거칠면(High Roughness) 빛이 사방으로 퍼져서 매트하게 보임.
* Normal Map: 실제 기하학적 메쉬(다각형)를 늘리지 않고도, 표면에 미세한 요철이나 주름, 질감(ex> 벽돌 틈새, 가죽 표면)이 있는 것처럼 빛을 왜곡시키는 위장용 맵

## Vertex Descriptor
정점 데이터 버퍼(`MTLBuffer`) 안에는 위치, 법선(방향), UV(텍스처 좌표) 같은 데이터가 한데 섞여서 들어올 수 있음. 이걸 GPU가 올바르게 잘라서 읽을 수 있도록 도와주는게 `MTLVertexDescriptor`

### 속성
* **Format**: 데이터 타입 설정 (ex> 위치 좌표 XYZ는 3개의 Float이라 `.float3` 매핑)
* **Offset**: 버퍼의 시작점으로부터 이 데이터가 몇 바이트 떨어져서 시작하는지 지정.
* **Buffer Index**: 0~31번까지 있는 GPU 버퍼 테이블 중 몇 번 인덱스에 이 데이터를 바인딩할 것인지 결정.
* **Stride**: 정점 한 세트의 전체 크기(byte). GPU가 다음 정점 위치를 읽기위해 메모리를 몇바이트 점프해야하는지에 대한 기준 스펙.

## Metal 좌표계와 NDC(Normalized Device Coordinate)
### 원점
3D 모델 자체의 중심점 `[0, 0, 0]`. Blender 정중앙에 만든 모델을 Metal에서 그대로 그리면 화면 한가운데 정점의 중심이 걸림.

### NDC 공간
화면 해상도가 4K든 아이폰 화면이든 상관없이, Metal이 내부적으로 사용하는 사각형의 고정 표준 상자 공간.
* **X축**: 좌측 `-1.0` ~ 우측 `1.0`
* **Y축**: 아래 `-1.0` ~ 위 `1.0`
* **Z축**: 화면 안쪽(앞) `0.0` ~ 화면 바깥쪽(뒤) `1.0`

#### 핵심 규칙 
GPU는 최종적으로 이 NDC 상자 공간 안에 들어오는 정점만 화면에 그려줌. 만약 모델의 바퀴가 공중에 떠 있다면, 정점 셰이더 함수에서 Y좌표 값을 빼서(`position.y -= 1.0`) 바닥(-1.0 위치)으로 강제로 내려주어야 함.

## Submesh와 Winding Order
### Submesh
모델에 여러 머터리얼(색상 그룹)이 지정되어 있다면, 각 색상 덩어리마다 정점들의 인덱스를 따로 모아둔 Submesh buffer가 존재.

#### 주의할 점
모델을 완전히 다 그리려면 단순히 드로우 콜을 한 번만 던지면 안 됨. `mesh.submeshes` 배열을 반복문으로 돌면서 모든 서브메시를 각각 드로우 콜 명령을 실행해줘야 GPU가 모델 전체를 그림.

### Winding Order
감기 순서. 정점들을 어떤 순서(방향)로 연결해서 삼각형을 만드는지에 대한 규칙.
* **Counter-Clockwise**: 반시계 방향. 정점이 반시계 방향으로 감기면 GPU는 이 삼각형을 카메라를 바라보는 앞면으로 인식.
* **Clockwise**: 시계 방향으로 감기면 카메라를 등지고 있는 뒷면으로 인식
* **Back-face Culling**: GPU는 효율적인 연산을 위해 어차피 보이지 않는 뒷면 삼각형들을 계산에서 제외하는 Culling을 수행. 따라서 정점 버퍼를 구성할 때 감기 순서를 올바르게 맞추는 것이 렌더링 최적화의 핵심

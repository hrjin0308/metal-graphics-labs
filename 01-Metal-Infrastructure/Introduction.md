# Introduction

## 1. Metal이란?
`Metal`(이하 메탈)은 그래픽 처리 장치(GPU) 제어를 위한 Apple의 **통합형, 저수준(Low-level), 저오버헤드(Low-overhead)** 애플리케이션 프로그래밍 인터페이스(API).

* **통합형 (Unified)**: 3D 그래픽스 렌더링과 데이터 병렬 컴퓨팅(Compute) 패러다임 모두에 동일한 인터페이스 적용
* **저수준 (Low-level)**: 개발자가 GPU 하드웨어 하부 구조에 직접 접근할 수 있는 API 지원
* **저오버헤드 (Low-overhead)**: 멀티스레딩 최적화 및 리소스 사전 컴파일(Pre-compiling)을 통한 실행 시점(Runtime) 비용 최소화

단순 기술적 정의를 넘어 Apple 실리콘 GPU의 강력한 병렬 처리 능력을 활용해 데이터를 시각화하거나 복잡한 수치 문제를 해결하는 가장 효율적인 수단. 현재 그래픽 렌더링 외에도 **머신러닝(ML), 이미지 및 영상 처리(Vision)** 등 다양한 분야에 최적화되어 활용됨.

## 2. Metal의 탄생 배경과 발전 과정

### 2.1 하드웨어 제어 API의 변화
* 과거 GPU 성능 활용을 위한 선택지는 `OpenGL`과 Windows 전용인 `DirectX`로 제한됨.
* 하드웨어 발전 속도 대비 기존 API들의 오버헤드가 비대해짐에 따라, 2013년 AMD에서 오버헤드를 줄이고 제어권을 높인 `Mantle`(맨틀) 프로젝트 발표.
* 맨틀은 기존 API 대비 최대 9배의 드로우 콜(Draw Call) 성능을 예고했으며, 그래픽과 연산 작업을 병렬 처리하는 **비동기 명령 큐(Asynchronous Command Queues)** 도입.
* 맨틀 자체는 주류 API 안착 전 중단되었으나, 차세대 로우레벨 API(`Vulkan`, `DirectX 12`, `Metal`)의 탄생에 직접적인 영감을 줌.

### 2.2 Metal의 연혁 및 버전별 주요 특징
Apple은 2014년 자체 로우레벨 API인 메탈을 공개한 이후, 하드웨어 아키텍처 성장에 맞춰 지속적으로 기능 확장 중.

## 3. 왜 Metal을 사용해야 할까?

메탈은 강력한 그래픽 파이프라인을 구동할 수 있는 최상급 그래픽스 API로, 글로벌 게임 엔진 및 주요 프로덕션 크리에이티브 앱들의 코어 기술로 활용됨.

### 3.1 상용 엔진의 한계 극복 (Unity & Unreal Engine)
* 유니티와 언리얼 엔진은 크로스 플랫폼 개발에 유리하나, 메탈의 최신 독점 기능을 신속하게 반영하지 못하는 한계 존재.
* 예: 유니티 내 테셀레이션(Tessellation) 지원 지연, 메시 셰이더(Mesh Shaders) 미지원 등.
* **Apple 실리콘 하드웨어 성능 최적화 및 최첨단 메탈 기능의 즉각적인 도입을 위해서는 로우레벨 단에서의 직접 제어가 필수적임.**

### 3.2 Metal 활용 주요 그래픽스 성공 사례
* **디비니티: 오리지널 신 2**: 라리안 스튜디오가 Apple과 협업하여 메탈 및 Apple GPU 하드웨어를 극한으로 활용, iPad 이식에 성공한 AAA 타이틀.
* **더 위트니스 (The Witness)**: 메탈 기반의 **자체 커스텀 엔진(Custom Engine)** 제작작. 디바이스 제약을 극복하고 데스크톱 버전과 동일한 퀄리티 구현.
* **기타 타이틀**: 힛맨, 바이오쇼크, 데우스 엑스, 마피아, 스타크래프트, 월드 오브 워크래프트, 포트나이트, 배트맨, 마인크래프트 등 다수의 메이저 타이틀이 메탈 기반 구동.

### 3.3 그래픽 가속 기반 프로덕션 앱
* **프로크리에이트 (Procreate)**: 메탈 아키텍처 전환 후 기존 대비 **4배의 렌더링 속도** 달성.
* **픽셀메이터 (Pixelmator)**: 메탈 2 기반 페인팅 엔진 및 다이내믹 페인트 블렌딩 기술 구현.
* **어피니티 포토 (Affinity Photo)**: 초고해상도 이미지 및 수천 개의 레이어가 포함된 복잡한 그래픽 합성 작업을 오버헤드 없이 부드럽게 처리.
* **머신러닝 및 딥러닝**: 메탈 하위 프레임워크인 **MPS**를 통해 합성곱 신경망(CNN)을 비롯한 딥러닝 모델 연산 가속화.

## 4. 언제 Metal을 사용해야 할까?

### 4.1 CPU와 GPU의 구조적 차이
플린의 분류학(Flynn’s taxonomy) 기준, GPU는 **SIMD(Single Instruction Multiple Data)** 연산 구조에 해당.

* **CPU (지연 시간 최적화)**: 단일 데이터를 빠르게 처리하는 순차(Serial) 처리에 특화. 소수의 강력한 코어로 구성.
* **GPU (처리량 최적화)**: 단위 시간당 대량의 데이터 처리 최적화. 수천 개의 작은 코어를 활용해 대규모 행렬 곱셈 등의 데이터를 **동시에 병렬(Parallel) 연산**.

> 💡 일반적인 프로그램은 직렬 루프(`입력 ➔ 처리 ➔ 출력`)로 작동하나, 매 프레임 수백만 픽셀의 연산이 필요한 그래픽스 영역에서는 대규모 병렬 처리에 특화된 GPU와 메탈 API가 필수적임.

### 4.2 Metal 선택을 위한 체크리스트
* 3D 모델을 하드웨어 수준에서 가장 효율적이고 빠르게 렌더링하고자 할 때
* 커스텀 라이팅 및 셰이딩을 구현하여 고유의 그래픽 스타일을 확보하고자 할 때
* 매 프레임 전 화면의 픽셀 색상을 계산하는 고부하 이미지/영상 처리가 필요할 때
* 과학적 시뮬레이션 등 거대한 수치 문제를 독립적 서브 문제로 분할해 병렬 처리하고자 할 때
* 딥러닝 모델 훈련 등 대규모 데이터셋을 동시에 병렬 연산해야 할 때
